# 修正内容と車でのテスト手順（2026/06/10）

## 今回なにを直したか（要点だけ）

### 1. ヴォクシーがつながらなかった原因への対策【最重要】
H16年式ヴォクシー(AZR60G)は、一般的なOBD2の通信方式ではなく
**トヨタ独自の「M-OBD（K-Line）」** という方式を使っています。
いままでのアプリは一般的な方式(ATSP0/4/5/6)しか試していなかったため、
車側(ECU)が一切応答せず `BUS INIT: ERROR` になっていました。

今回、トヨタ車で実績のある専用の初期化コマンドを組み込みました:

```
ATIB96     → 通信速度を9600bpsに
ATIIA13    → 呼びかけ先アドレスを 13 に
ATSH8113F1 → トヨタECU用のヘッダー設定
ATSPA4     → KWP(5ボーイニシャル)方式+自動フォールバック
ATSW00     → ウェイクアップ停止
```

これはTorque Proという有名アプリでトヨタの同年代の車
（シエンタ・ウィッシュ・ランクル100など）を動かすときに使われている設定と同じものです。

### 2. 「全自動」モードを追加（これを選べばOK）
プロトコル選択に **「全自動（順番に試す・推奨）」** を追加しました。
①トヨタM-OBD → ②標準OBD2 → ③CAN の順に自動で試します。
ヴォクシーでもフェラーリでも、これを選んでおけば大丈夫です。

### 3. 通信の作りを全面的に丈夫にした
- 以前は「コマンドを送って一定時間待つだけ」だったのを、
  **アダプターの応答(>)をちゃんと待ってから次へ進む** 方式に変更
- K-Line初期化は応答に5秒以上かかるため、初回は**20秒まで待つ**ように変更
  （以前は待ちが短すぎて、交渉中に失敗扱いになっていた可能性があります）
- 応答が来ないとデータ取得が永久に止まってしまうバグを修正
- アクセル開度(0111)に応答しない車では、自動で「回転数のみモード」に切替
  （音はちゃんと鳴ります。音量は回転数から自動計算）

### 4. テスト用の音をエンジン風に
いままでの「ピーーー」という音から、倍音を重ねたエンジン風の音に変更しました。
（本物のフェラーリ音への差し替え方法は「GitHubでビルドする手順.md」の最後を参照）

## 車でのテスト手順

1. **新しいAPKをスマホに入れる**
   GitHubの Actions タブ → 最新の「Build APK」→ ferrari-sound-apk をダウンロード
   → スマホに転送してインストール（上書きでOK）

2. **車に乗って、エンジンをかける**
   ※キーOFFだとECUに電源が入らず100%失敗します。**必ずエンジン始動後に**。

3. iCar ProをOBD2ポートに挿し、スマホとBluetoothペアリング済みであることを確認

4. アプリを起動 → プロトコルは **「全自動（順番に試す・推奨）」** のまま
   → ICARPRO をタップ

5. **1〜2分待つ**（全自動モードは複数の方式を順に試すので時間がかかります）
   画面下の通信ログに進捗が出ます。

## 結果の見かた（画面下の通信ログ）

| ログ表示 | 意味 |
|---|---|
| `=== 接続成功: トヨタ K-Line (M-OBD) ===` | 🎉 成功。回転数が表示され音が鳴ります |
| `41 0C ...` という応答 | ECUから回転数データが来ています |
| `UNABLE TO CONNECT` / `BUS INIT: ERROR` が全方式で出る | 下の「それでもダメな場合」へ |

## それでもダメな場合

通信ログ画面を**スクリーンショット**して、Claudeに見せてください。
特に「トヨタ K-Line (M-OBD) 試行中」のときに何が返ってきたかが重要です。

考えられる残りの原因:
- アダプターがELM327 **v2.1クローン**の場合、K-Lineのタイミング制御に
  非対応のことがある（iCar Proは大丈夫なはずですが、ログの `ATZ` の応答に
  表示されるバージョンを確認）
- `ATIB96` の応答が `?` の場合 → アダプターがこのコマンド非対応。要相談
- OBD2ポートの7番ピン(K-Line)が結線されていない激安延長ケーブル等を
  使っている場合は直挿しで試す
